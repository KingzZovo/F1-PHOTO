## What "v1.5.0 done" looks like

1. **Steps A / B / C тЬЕ complete** тАФ real ONNX pipeline is the only legal path; any fallback is a WARN; smoke fails on any WARN.
2. Milestone #1 (gradual cut-over) shipped тАФ `F1P_INFERENCE_STUB_FALLBACK` is removed from smoke L130 and from any deployment manifest; `stub_fallback_enabled()` becomes a tombstone (or is deleted entirely once we are confident).
3. Milestone #2 (real-dataset smoke baseline) shipped тАФ at least one curated real-photo dataset is checked in (or referenced) and a `make real-smoke` (or equivalent) target reproduces the recognition_items distribution.
4. Milestone #3 (precision / recall baseline) shipped тАФ P/R numbers per detect_target are recorded in `docs/recognition_pipeline.md`, and `Thresholds::DEFAULT` is either confirmed or adjusted with a documented reason.

Android client (#4) remains deferred per King's standing direction; we do not gate v1.5.0 on it.
