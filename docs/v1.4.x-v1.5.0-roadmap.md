# v1.4.x → v1.5.0 Inference Real-Pipeline Rollout Roadmap

This roadmap tracks the cut-over from the env-gated stub fallback (v1.4.x) to a
real ONNX inference pipeline locked in by default (v1.5.0). It is the
authoritative status board for Steps A / B / C and the follow-on milestones.

- 🟡 next — accepted as the immediate follow-on after Step C; not yet started.

## What "v1.5.0 done" looks like

1. **Steps A / B / C ✅ complete** — real ONNX pipeline is the only legal path; any fallback is a WARN; smoke fails on any WARN.
2. Milestone #1 (gradual cut-over) shipped — `F1P_INFERENCE_STUB_FALLBACK` is removed from smoke L130 and from any deployment manifest; `stub_fallback_enabled()` becomes a tombstone (or is deleted entirely once we are confident).
3. Milestone #2 (real-dataset smoke baseline) shipped — at least one curated real-photo dataset is checked in (or referenced) and a `make real-smoke` (or equivalent) target reproduces the recognition_items distribution.
4. Milestone #3 (precision / recall baseline) shipped — P/R numbers per detect_target are recorded in `docs/recognition_pipeline.md`, and `Thresholds::DEFAULT` is either confirmed or adjusted with a documented reason.
5. Milestone #2a (owner-known person bootstrap path) shipped — ✅ done at commit 9a85f7c. Milestone #2b (face fixture + matched-bucket smoke) — ✅ done at commit e84cdd0. Milestone #2a-tool (owner-known tool/device bootstrap, symmetric to #2a) — ✅ done at commit e37d6e4; the cold-start story is now end-to-end-proven for **person, tool, and device** owners. Same wo_raw cross-project recall path validates seeded gallery for all three owner kinds.

Android client (#4) remains deferred per King's standing direction; we do not gate v1.5.0 on it.
