## Sandbox / MCP quirks (carry-over from previous threads, all confirmed this thread)

- /bin/sh = dash, no `disown` — background long-running tasks via `setsid nohup bash -c '...' </dev/null >/tmp/X.log 2>&1 &`.
- MCP `run_command` single-call timeout ≤ 30s. For longer tasks (eval reruns, drains), background + poll.
- `read_text` / `list_files` paths resolve from `workspace_root=/root` not `session_cwd=/root/F1-photo` — always pass absolute paths.
- `apply_patch` paths also resolve from `/root` — use absolute paths.
- `apply_patch` cannot match within a mega-line — for large markdown lines (e.g. roadmap narrative paragraphs) use Python in-place: `pathlib.Path(p).read_text() → .replace(needle, replacement, 1) → .write_text(...)`.
- `git_commit` parameter is `paths` (not `files`).
- Python heredoc: shell eats `\'` — use raw heredoc `cat > /tmp/x.py <<'EOF' ... EOF` then `python3 /tmp/x.py`, or pass JSON via env var.
- MCP server has occasional transient `Failed to connect to MCP server`; one retry generally fixes it.
